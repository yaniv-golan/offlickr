# offlickr v0.1.0 — Foundation

**offlickr** turns a Flickr *Your Flickr Data* export into a self-contained static website.
This first release ships the ingest, derive, and render stages: you can now
point it at your Flickr zip files and get a browsable offline archive.

## What's included

- **Ingest** (`offlickr ingest`): parses all Flickr export JSON files
  (account, albums, galleries, groups, contacts, followers, faves, photos) into
  a validated `model.json` + JSON Schema. Handles zip auto-extraction with
  SHA-256 content keying.
- **Derive** (`offlickr derive`): generates WebP thumbnails (240px) and display
  images (1600px), copies originals, extracts EXIF, and emits a client-side
  search index.
- **Render** (`offlickr render` / `offlickr build`): produces a static site
  with a paginated photostream, per-photo detail pages (title, description,
  comments, tags, EXIF), and an about page. Vanilla HTML/CSS/JS — works from
  `file://`, a USB stick, or any static host.
- **`minimal-archive` theme**: white background, serif headings, justified
  photo grid. Client-side search (debounced, substring, NFC-normalized).
- **`offlickr serve`**: one-liner local preview server.

## Known limitations in v0.1.0

- Albums, galleries, tags, map, faves, groups, and testimonials pages are
  planned for v0.2. They are not rendered in v0.1.
- External asset fetching (user avatars, fave thumbnails) is planned for v0.9.
  Faves and external gallery photos render as text tombstones.
- The `flickr-classic` theme is planned for v0.2.
- Windows is not officially supported yet (day-of-month formatting uses a
  Unix-only strftime flag). Planned for v0.2.
- Videos require ffmpeg on PATH for poster thumbnail generation; without it a
  generic SVG placeholder is used.
