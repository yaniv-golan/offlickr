# offlickr v0.1.0 — Foundation

**offlickr** turns a Flickr *Your Flickr Data* export into a self-contained static website.
This first release ships all core stages — ingest, derive, render, and external asset fetch —
plus a complete set of archive pages.

## What's included

- **Ingest** (`offlickr ingest`): parses all Flickr export JSON files
  (account, albums, galleries, groups, contacts, followers, faves, photos) into
  a validated `model.json` + JSON Schema. Handles zip auto-extraction with
  SHA-256 content keying.
- **Derive** (`offlickr derive`): generates WebP thumbnails (240px) and display
  images (1600px), copies originals, extracts EXIF, and emits a client-side
  search index. Parallelised via `ProcessPoolExecutor`.
- **Render** (`offlickr render` / `offlickr build`): produces a complete static
  site — works from `file://`, a USB stick, or any static host:
  - Paginated photostream (60 photos/page)
  - Per-photo detail pages (title, description, comments, tags, EXIF, notes, geo minimap)
  - Albums, galleries, groups, faves, tags, testimonials, about, and geo map
  - Date archive (`archive/YYYY/` browsing)
  - `--flickr-origin` mode: rewrites internal links to point back to flickr.com
- **External asset fetch** (`offlickr fetch-external` / `--archive-external`):
  downloads avatars and fave thumbnails from the Flickr API into the archive.
  Requires a free Flickr API key; the core pipeline needs no key.
- **`minimal-archive` theme**: white background, serif headings, justified
  photo grid, bundled WOFF2 fonts. Client-side search (debounced, substring,
  NFC-normalized). Designed to look clean 20 years from now.
- **Privacy controls**: private photos excluded by default (`--include-private-photos`
  to include). Contacts, followers, Flickrmail, and comment views excluded by
  default (`--include-private` to include).
- **`offlickr serve`**: one-liner local preview server.
- **`offlickr inspect`**: CLI introspection of the parsed archive.

## Known limitations in v0.1.0

- The `flickr-classic` theme is not yet available.
- Videos require ffmpeg on PATH for poster thumbnail generation; without it a
  generic SVG placeholder is used.
